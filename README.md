# AuditPilot

감사자료 수집부터 클렌징, 분석, 테스트, 문서화까지 이어지는 반복 업무를 줄이고, 감사인이 판단에 더 집중할 수 있도록 만든 Streamlit 기반 데모 앱입니다.

AuditPilot의 목적은 감사인의 판단을 대체하는 것이 아닙니다. 자료 정리, 형식 통일, 검증, 이상치 후보 산출, 질의 문안과 조서 초안 작성처럼 반복적이고 시간이 많이 드는 업무를 줄여 감사인이 위험평가, 예외 해석, 회사 설명의 타당성, 최종 결론에 더 많은 시간을 쓰도록 돕는 것입니다.

## 문제의식

감사업무를 실제로 해보면 병목은 꽤 명확합니다.

- 전기 PBC 리스트를 거의 그대로 가져오면서 자료 요청 목적이 흐려집니다.
- 회사마다 GL, 계정명세서, 월보, 주석 backdata의 형식과 열 이름이 다릅니다.
- 엑셀 클렌징, 헤더 매핑, 차대변 대사, 명세서 대사에 많은 시간이 들어갑니다.
- 분석적검토에서는 전기 대비 증감, 월별 추이, 거래처별 변동을 반복적으로 확인해야 합니다.
- 질의 문안, 리뷰 답변, 조서 결론은 숫자와 논리가 맞게 문서화되어야 합니다.

이 앱은 위 병목을 하나의 흐름으로 묶어 보여주는 PoC입니다.

## 화면 흐름

현재 앱은 6개 탭으로 구성되어 있습니다.

1. **병목 진단**
   - 감사업무의 반복 업무와 Digital 활용 지점을 정리합니다.
   - Planning, Execution, Completion 단계별 비효율과 기대효과를 보여줍니다.

2. **PBC 요청**
   - 계정별 요청자료, 감사주장, 수행절차, 제출 형식을 연결합니다.
   - 계정 선택에 따라 고객사 요청 문구가 바뀝니다.
   - 매출, 매출채권, 현금및현금성자산, 재고자산, 유형자산, 매입채무, 미지급비용, 차입금, 매출원가, 판매관리비를 지원합니다.

3. **업로드·매핑**
   - 데모 샘플 또는 xlsx 4개를 불러옵니다.
   - 회사별로 다른 원장 헤더를 표준 스키마로 매핑합니다.
   - 매핑은 자동 확정되지 않고 감사인이 게이트 1에서 확정합니다.

4. **클렌징·검증**
   - 중복, 결측, 차대변 불일치, 기간 외 거래, 명세서 대사 차이를 검증합니다.
   - 예외는 삭제하거나 임의 수정하지 않고 감사인이 판단할 수 있도록 표시합니다.

5. **분석·테스트**
   - 전기 대비, 월별 추이, 거래처별 증감과 확인 필요 후보를 산출합니다.
   - Top 후보에 대해 후속 질의 문안을 생성합니다.
   - 질의 문안은 게이트 2에서 감사인이 승인합니다.

6. **문서화·판단**
   - 분석 결과를 바탕으로 감사조서 초안을 생성합니다.
   - 문서 내 숫자가 원천 계산값과 일치하는지 검증합니다.
   - 잠정결론은 게이트 3에서 감사인이 승인합니다.
   - 승인 후 xlsx 또는 Markdown으로 export할 수 있습니다.

## 주요 기능

- 주요 계정별 PBC 요청 템플릿
- 업로드 자료 종류 판별
- 원장 헤더 매핑과 표준 스키마 정규화
- 결정적 검증 룰 5종
- 전기 대비, 월별, 거래처별 분석
- 확인 필요 후보 Top 추출
- 고객사 후속 질의 문안 생성
- 조서 초안 생성
- 문서 내 수치 전수 검증
- xlsx, Markdown export
- 화면 우측 하단 플로팅 챗봇 `삼일이`
- GPT API 기반 감사업무 질의응답
- Streamlit UI 기반 human-in-the-loop 흐름
- Pytest 기반 단위, 통합, UI 흐름 테스트

## 삼일이 챗봇

앱 우측 하단에는 플로팅 챗봇 `삼일이`가 있습니다.

- 마우스로 자유롭게 위치를 옮길 수 있습니다.
- `X`를 누르면 말풍선만 숨기고 캐릭터는 남깁니다.
- 캐릭터를 다시 누르면 말풍선이 열립니다.
- PBC 요청, 자료 클렌징, 분석적검토, 테스트 설계, 조서 초안, 감사인 판단 지점에 대해 질문할 수 있습니다.

챗봇은 OpenAI-compatible GPT API와 연결되어 있습니다. `OPENAI_API_KEY`가 설정되어 있으면 실제 AI 응답을 사용하고, 키가 없거나 사용량 한도 문제가 있으면 앱 안에서 안내 메시지를 보여줍니다.

## 감사인 판단이 남는 지점

AuditPilot은 업무를 보조하지만 결론을 대신 내리지 않습니다.

- 자동 매핑 결과는 감사인이 열 의미를 확인한 뒤 확정합니다.
- 검증 예외는 오류인지 정상 거래인지 감사인이 판단합니다.
- 분석 후보는 추가 확인 대상일 뿐, 왜곡표시나 감사의견 판단이 아닙니다.
- 질의 문안은 감사인이 필요성과 범위를 검토한 뒤 승인합니다.
- 조서 초안은 Draft이며, 최종 결론은 감사인이 승인합니다.

## 실행 방법

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/streamlit run app.py
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:8501
```

### GPT API 연결

프로젝트 루트에 `.env` 파일을 만들고 아래 값을 넣습니다.

```env
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini
OPENAI_BASE_URL=https://api.openai.com/v1
```

`.env`는 Git에 올라가지 않도록 제외되어 있습니다. 실제 API 키는 GitHub에 커밋하지 않습니다.

키 설정 후 앱을 재시작하면 `삼일이` 챗봇이 GPT API로 답변합니다.

## 추천 데모 순서

1. `③ 업로드·매핑` 탭을 엽니다.
2. `데모 샘플 불러오기`를 누릅니다.
3. `게이트 1 · 매핑 확정`을 누릅니다.
4. `④ 클렌징·검증`에서 `검증 실행`을 누릅니다.
5. `⑤ 분석·테스트`에서 `분석 실행`을 누릅니다.
6. `Top 1 후속 질의 생성` 후 `게이트 2 · 질의 문안 승인`을 누릅니다.
7. `⑥ 문서화·판단`에서 `조서 초안 생성` 후 `게이트 3 · 잠정결론 승인`을 누릅니다.
8. 조서 패키지를 xlsx 또는 Markdown으로 내려받습니다.

## 테스트

```bash
.venv/bin/pytest -q
```

테스트 범위는 다음을 포함합니다.

- 자료 ingest와 문서 종류 판별
- 헤더 매핑과 정규화
- 데이터 검증 룰
- 분석 후보 점수화
- 생성 문안의 수치 검증
- 조서 export
- Streamlit 전체 승인 게이트 흐름

## 프로젝트 구조

```text
app.py                         Streamlit UI와 승인 게이트
auditpilot/core/               데이터 처리, 검증, 분석, 문서 검증
auditpilot/data/               데모 샘플 데이터 생성
auditpilot/llm/                Fixture/OpenAI-compatible LLM 클라이언트
auditpilot/harness/            내부 평가용 harness 코드
config/                        헤더 매핑 alias 설정
fixtures/                      캐시 응답과 오류 조서 fixture
tests/                         단위, 통합, export, UI 테스트
docs/                          기획서, 설계서, QA 보고서
```

## 한계

- 샘플 데이터는 기능 검증용 가상 데이터입니다.
- 앱은 확인 필요 후보를 제시할 뿐 왜곡표시 여부나 감사의견을 판단하지 않습니다.
- GPT API 키가 없거나 계정 사용량 한도, 결제 상태 문제가 있으면 챗봇은 실제 AI 응답을 받을 수 없습니다.
- 거래 수준 증빙 매칭, P/Q 분해, 조회서 회신 관리, 주석 상호참조 자동 검토는 현재 범위 밖입니다.
- 실제 업무 적용 시 회사별 계정 구조, 중요성, 산업 특성, 내부 정책에 맞춘 검증 룰 보강이 필요합니다.
