## 2. 실제 브라우저 QA

로컬 Streamlit 서버에서 다음 경로를 직접 클릭했다.

```text
샘플 로드 → 매핑 확정 → 검증 5종 → 분석 → 질의 생성·승인
→ 조서 생성 → 수치 검증 → 잠정결론 승인 → Export 활성화
```

확인 결과:

- 검증 화면에서 중복·결측·차대변·기간·대사 예외가 각각 표시됨
- 후속 질의의 `280,000,000원`, `+180.0%`가 원천 계산값과 일치함
- 조서에 DRAFT 표기와 판단 유보 문구가 표시됨
- Review 검증기 통과 전에는 게이트 3이 비활성화됨
- 게이트 3 승인 후 xlsx·Markdown 버튼이 모두 활성화됨
- 생성된 가상 xlsx 4종을 브라우저에서 동시에 업로드하고 헤더 시그니처 기반으로 당기·전기 원장, 시산표, 명세서가 분류됨
- 브라우저 콘솔에서 애플리케이션 코드 오류 없음

## 3. QA 중 발견·수정한 결함

동일한 표시 금액을 가진 다른 후보의 근거 키가 조서 각주에 함께 연결되는 추적성 결함을 발견했다. 조서가 선택된 Top 1 후보의 키만 참조하도록 수정하고 회귀 테스트를 추가했다. 수정 후 브라우저에서 근거 키 2개만 남는 것을 재확인했다.

## 4. 미실행 항목과 사유

- 클라우드 LLM vs Ollama 실모델 비교: 현재 환경에 API 키와 Ollama가 없어 미실행
- 실제 고객 데이터: 비밀유지와 개인정보 보호를 위해 사용하지 않음
- 외부 이메일 발송: 제품 범위에서 의도적으로 제외

위 항목은 fixture 결과로 대체하거나 통과한 것으로 표현하지 않는다.
