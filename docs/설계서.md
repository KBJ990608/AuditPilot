# AuditPilot 상세 설계서

| 항목 | 내용 |
|---|---|
| 버전 | v1.0 |
| 작성일 | 2026-07-22 |
| 상태 | 확정 |
| 기준 | 숫자는 결정적 코드가, 문장은 LLM이, 결론은 감사인이 만든다 |

## 1. 설계 개요

AuditPilot은 Streamlit UI, 결정적 계산 계층(`core`), 문안 생성 계층(`llm`)으로 분리한다. `llm`은 원본 DataFrame을 받지 않고 코드가 만든 문자열 변수만 받는다. LLM 출력은 수치 검증을 통과해야 표시되며 감사인의 승인 없이는 매핑 확정, 질의 확정, 조서 Export가 불가능하다.

```text
Streamlit UI → ingest/mapping → validate → analytics
                                    ↓
                     ComputedValue registry (single source of truth)
                                    ↓
                         LLM 문안 → reviewer → HITL → export
```

의존 방향은 `app → core/llm`, `harness → llm/core.reviewer`로 단방향이다. `core`는 `llm`을 import하지 않는다.

## 2. 데이터 계약

### 2.1 자료 종류와 표준 스키마

```python
from enum import StrEnum

class DocumentType(StrEnum):
    CURRENT_GL = "당기원장"
    PRIOR_GL = "전기원장"
    TRIAL_BALANCE = "시산표"
    AR_SUBLEDGER = "매출채권명세서"
    UNKNOWN = "미분류"
```

원장 필수 컬럼은 `전표일자(datetime64[ns])`, `전표번호(string)`, `계정코드(string)`, `계정명(string)`, `거래처(string)`, `차변(int64)`, `대변(int64)`, `적요(string)`, `전표유형(string)`이다. 분석용 파생 컬럼은 `연도`, `월`, `순액=대변-차변`, `원본행`이다. 시산표는 `계정코드`, `계정명`, `차변잔액`, `대변잔액`; 명세서는 `거래처`, `잔액`을 사용한다. 합계행은 정규화 전 분리한다.

자료 판별은 헤더 정규화 후 다음 시그니처의 포함률로 결정한다. 원장 `{전표일자,전표번호,차변,대변}` 75% 이상, 시산표 `{계정코드,차변잔액,대변잔액}` 100%, 명세서 `{거래처,잔액}` 100%다. 원장 연도는 데이터 최빈연도로 당기/전기를 구분하며 목표연도와 같으면 당기다. 동률 또는 임계치 미달은 `UNKNOWN`으로 반환하여 사용자가 지정한다.

### 2.2 열 매핑

`config/header_aliases.json`은 `{canonical: [aliases...]}` 구조다. 헤더는 공백·괄호·밑줄을 제거하고 소문자화한다. 정확 별칭 → `difflib.SequenceMatcher` 유사도 0.82 이상 → 미매핑 순으로 처리한다. 0.82 미만은 자동 확정하지 않는다.

LLM 제안은 아래 JSON만 허용한다.

```json
{"suggestions":[{"source":"Amt(CR)","target":"대변","confidence":0.91,"reason":"credit amount 약어"}]}
```

확정 결과는 `artifacts/mappings/{sha256(file_bytes)[:12]}.json`에 원본 헤더, 매핑, 확정시각을 저장한다.

### 2.3 검증 결과

```python
@dataclass(frozen=True)
class ValidationItem:
    rule_id: str
    passed: bool
    message: str
    exception_rows: tuple[int, ...] = ()
    expected: int | None = None
    actual: int | None = None

@dataclass(frozen=True)
class ValidationReport:
    items: tuple[ValidationItem, ...]
    passed: bool
```

룰은 `V01_DUPLICATE`(전표번호·차변·대변 동일), `V02_REQUIRED`(필수값 결측), `V03_BALANCE`(전표번호별 차변합계=대변합계), `V04_PERIOD`(목표연도 밖), `V05_RECONCILE`(명세서 합계=시산표 매출채권 잔액)이다. 금액 허용오차는 0원이다. 중복은 삭제하지 않고 예외 표시 후 사용자가 제외 여부를 결정한다.

### 2.4 분석 결과와 점수

```python
@dataclass(frozen=True)
class Candidate:
    candidate_id: str
    rank: int
    category: str
    entity: str
    score: int
    assertions: tuple[str, ...]
    evidence: dict[str, int | float | str]
    source_rows: tuple[int, ...]

@dataclass(frozen=True)
class AnalyticsResult:
    account_change: pd.DataFrame
    monthly_trend: pd.DataFrame
    customer_change: pd.DataFrame
    new_customers: tuple[str, ...]
    dormant_customers: tuple[str, ...]
    candidates: tuple[Candidate, ...]
```

PoC 수행중요성은 기본 50,000,000원이며 UI에서 변경 가능하다. 후보 점수(최대 100)는 증감액이 수행중요성 이상 30점, 증감률 절대값 50% 이상 25점, 100% 이상 추가 10점, 12월 집중률 25% 이상 20점, 신규 15점, 기말 5일 거래 10점이다. 상위 5개를 `score desc, abs(change) desc, entity asc`로 정렬한다. 0으로 나누는 경우 전기 0·당기 비0은 증감률을 `None`, 신규 플래그로 처리한다.

### 2.5 수치 레지스트리와 조서

```python
@dataclass(frozen=True)
class ComputedValue:
    key: str                 # 예: customer.daesung.current_sales
    value: int | float
    display: str             # 예: "280,000,000원", "+180.0%"
    source: str              # 예: "당기원장 원본행 501~520"

@dataclass
class Workpaper:
    title: str
    objective: str
    source_documents: list[str]
    procedures: list[str]
    fluctuations_markdown: str
    management_explanation: str
    follow_ups: list[str]
    provisional_conclusion: str
    referenced_keys: list[str]
    approved: bool = False
```

LLM 프롬프트에는 `display` 문자열만 주입한다. 생성문에 등장 가능한 수치는 `referenced_keys`의 display 토큰뿐이다. 잠정결론은 “현재까지 확인된 변동 후보에 대해 회사 설명 및 증빙의 추가 확인이 필요하며, 본 초안만으로 감사 결론을 형성하지 않는다.”로 고정한다.

### 2.6 캐시와 골든셋

Fixture는 프롬프트 해시 대신 안정적인 업무 키(`pbc_ar`, `mapping_unknown_headers`, `candidate_daesung`, `query_daesung`, `workpaper_ar`)로 `fixtures/llm_responses.json`에 저장한다. 프롬프트 문구 변경에도 데모가 깨지지 않고, 어떤 응답인지 사람이 감사할 수 있기 때문이다. 각 값은 `{provider, model, cached, content}` 구조다.

골든셋 JSONL은 `id`, `task`, `input`, `expected`, `forbidden_phrases`를 갖는다. 결과는 `provider`, `model`, `prompt_version`, `latency_ms`, `scores`, `raw_output`을 기록한다.

## 3. 모듈별 상세 설계

```python
# core/ingest.py
def read_tabular(source: str | Path | BinaryIO, sheet_name: str | int = 0) -> pd.DataFrame: ...
def normalize_header(value: object) -> str: ...
def classify_document(df: pd.DataFrame, target_year: int = 2025) -> DocumentType: ...

# core/mapping.py
def propose_mapping(headers: Sequence[str], aliases: Mapping[str, Sequence[str]], threshold: float = .82) -> MappingResult: ...
def apply_mapping(df: pd.DataFrame, mapping: Mapping[str, str]) -> pd.DataFrame: ...
def normalize_gl(df: pd.DataFrame) -> pd.DataFrame: ...
def save_confirmed_mapping(mapping: MappingResult, file_hash: str, directory: Path) -> Path: ...

# core/validate.py
def validate_gl(df: pd.DataFrame, target_year: int) -> tuple[ValidationItem, ...]: ...

# core/analytics.py

def build_computed_values(result: AnalyticsResult) -> dict[str, ComputedValue]: ...

# core/workpaper.py
def build_workpaper(result: AnalyticsResult, narrative: str, registry: Mapping[str, ComputedValue]) -> Workpaper: ...

# core/reviewer.py

# core/export.py
def export_xlsx(workpaper: Workpaper, candidates: Sequence[Candidate], report: ValidationReport) -> bytes: ...
def export_markdown(workpaper: Workpaper, registry: Mapping[str, ComputedValue]) -> bytes: ...
```

금액 정규식은 `(?<!\d)[+-]?\d{1,3}(?:,\d{3})*(?:\.\d+)?\s*(?:원|천원|백만원|억원)?`, 비율은 `(?<!\d)[+-]?\d+(?:\.\d+)?\s*%`, 건수는 `(?<!\d)\d+\s*건`을 사용한다. 날짜·회계연도·후보 순위는 검증 대상에서 제외한다. 단위는 원=1, 천원=1,000, 백만원=1,000,000, 억원=100,000,000으로 정규화한다. `ReviewResult.passed`는 추출 토큰이 허용 display 값과 모두 일치하고 금지 표현이 없을 때만 참이다.

`LLMClient.generate(task_key, system, user) -> LLMResponse`를 공통 인터페이스로 둔다. `FixtureClient`는 기본 provider이며 네트워크를 쓰지 않는다. `OpenAICompatibleClient`는 `base_url`, `api_key`, `model`, `timeout=20`을 받아 `/chat/completions`를 호출한다. 실패 시 앱 계층이 fixture로 폴백하고 배지를 표시한다. 원본 원장은 API에 전달하지 않는다.

## 4. 화면 및 상태 설계

탭은 `① PBC 요청`, `② 업로드·매핑`, `③ 검증`, `④ 분석`, `⑤ 조서·Export`, `⑥ Harness`다. 상태 키는 `documents`, `document_types`, `mapping_draft`, `mapping_confirmed`, `normalized`, `validation_report`, `analytics_result`, `registry`, `query_text`, `query_review`, `query_approved`, `workpaper`, `workpaper_review`, `workpaper_approved`, `llm_provider`다.

상태 전이는 순방향이며 상위 입력 변경 시 하위 상태를 모두 폐기한다.

- 게이트 1: 필수 컬럼이 모두 매핑되고 사용자가 “매핑 확정”을 눌러야 `mapping_confirmed=True`; 검증 버튼 활성화.
- 게이트 2: 질의 수치 검증 통과 후 사용자가 문안을 확인해야 `query_approved=True`. 외부 발송 기능은 없다.
- 게이트 3: 조서 수치 검증 및 금지표현 검사 통과 후에만 승인 버튼 활성화. 승인 후 Export 버튼 활성화.

차트는 Streamlit 기본 차트로 월별 당기/전기 매출 막대와 거래처별 증감액 Top 10 가로막대만 제공한다. 차트 원본 테이블을 함께 표시한다.

## 5. LLM 프롬프트 계약

모든 system prompt에는 다음을 공통 삽입한다: “제공된 사실과 수치만 사용하라. 새 수치, 감사의견 또는 단정적 결론을 생성하지 마라. 지정 JSON 이외의 텍스트를 출력하지 마라.”

| 작업 | 입력 | 출력 키 |
|---|---|---|
| 열 매핑 | 미지 헤더와 canonical 목록 | `suggestions[]` |
| 후보 해석 | category, entity, 허용 display 값 | `narrative`, `used_keys[]` |
| 질의 문안 | 후보 사실과 허용 display 값 | `question`, `used_keys[]` |
| 조서 서술 | Top 5 사실과 허용 display 값 | `narrative`, `used_keys[]` |

JSON parse 실패, 알 수 없는 `used_keys`, 금지 표현, 허용되지 않은 숫자 중 하나라도 있으면 출력은 폐기하고 fixture로 폴백한다.

## 6. 샘플 데이터 설계

`data/make_sample.py`는 `random.Random(20250722)`를 사용해 가상회사 `(주)한빛유통`의 FY2024 550행, FY2025 600행 원장을 생성한다. 거래처는 `대성물산`, `새롬리테일`, `한강마트`, `동서상사`, `미래유통`, `청운상회`, `가온마켓`, `중앙유통`, `세진상사`, `태양마트`, `명진유통`, `우리상회`, `삼우물산`, `제일마트`, `백두상사`, `한빛리테일`, `성진유통`, `대양상사`, `보람마트`, `정우물산`이다.

당기 raw 헤더는 `Posting Dt`, `Voucher No`, `Acct Cd`, `계정과목`, `Customer`, `Debit Amount`, `Amt(CR)`, `Description`, `Entry Type`으로 한다. `Posting Dt`와 `Amt(CR)`은 별칭 사전에 넣지 않아 fixture LLM 제안 경로를 시연한다.

주입 조건은 다음과 같다.

1. 동일 전표번호·금액 행 2건 복제.
2. 거래처 공란 1건.
3. 명세서 합계가 시산표 매출채권 잔액보다 정확히 3,000,000원 큼.
4. 2026-01-03 전표 1건.
5. 대성물산 당기 12월 280,000,000원, 전기 동월 100,000,000원(+180%), 당기 중 70%를 12/28~31에 배치.
6. 신규 `새롬리테일`은 2025년 7월부터 등장, 휴면 `정우물산`은 2024년에만 존재.
7. 2025년 12월 총매출은 1~11월 평균의 2.5배가 되도록 보정.
8. `fixtures/invalid_workpaper.md`에는 레지스트리 값과 1,000,000원 차이 나는 수치 1건을 둔다.

생성 후 assertion으로 8개 조건을 자체 검증하고 4개 xlsx를 `data/samples/`에 쓴다.

## 8. 구현 순서와 완료 조건

1. 패키지 구조·타입·샘플 생성기
2. 위 테스트를 먼저 작성하고 실패 상태 확인
3. ingest/mapping/validate 구현
4. analytics/reviewer/workpaper/export 구현
5. fixture/OpenAI-compatible LLM 계층과 하네스 구현
6. Streamlit UI 및 상태 게이트 구현
7. pytest 전체 통과, 샘플 이상 7종 및 오류 조서 차단 확인
8. 앱 브라우저 E2E 5회, Export 파일 실제 재개방

완료 조건은 기획서의 성공 기준 5개와 동일하다. 실모델 2종 결과는 실제 모델 실행 환경에서만 생성하며, fixture 결과를 실모델 결과로 표기하지 않는다.
